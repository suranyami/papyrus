## Decisions Made

- `pin_config` is a flat atom-keyed map (`%{atom() => non_neg_integer()}`) — chosen per D-01 from CONTEXT.md for simplicity and debuggability; keys mirror `DEV_Config.h` naming
- `pin_config` in `@enforce_keys` — per D-02; every display module must provide GPIO config explicitly
- Tests use `ArgumentError` not `KeyError` for struct enforcement — Elixir's `struct!/2` raises `ArgumentError` when enforce_keys are missing (plan's behavior spec said `KeyError` which was incorrect)
