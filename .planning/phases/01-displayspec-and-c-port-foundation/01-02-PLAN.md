<objective>
Harden `c_src/epd_port.c` with `select()`-based stdin polling so the OS process detects stdin EOF (BEAM death) during idle waits and exits cleanly. This prevents zombie `epd_port` processes that hold the SPI/GPIO device busy.

Purpose: DRIVER-01 requires the C port to never leave zombie processes. Currently the port only detects EOF when `read_exact()` fails mid-read. If the BEAM dies while the port is blocked waiting for the next command header, the port hangs until the next read attempt. Adding `select()` before `read_exact()` catches EOF during idle.

    Modify `c_src/epd_port.c` to add `select()` monitoring of stdin before the blocking `read_exact()` call in the main loop. This is the simplest approach per Claude's Discretion: wrap the idle wait only, not busy-pin polling loops.

    1. Add `#include <sys/select.h>` to the includes at the top of the file (after `#include <unistd.h>`).

    2. Define a select timeout constant near the other `#define` lines:
       ```c
       #define STDIN_POLL_TIMEOUT_SEC 1  /* select() timeout for stdin EOF detection */
       ```

    3. Replace the main loop's direct `read_exact(hdr, 5)` call with a select-then-read pattern:

       ```c
       for (;;) {
           /* Poll stdin for readability — detects EOF even when idle */
           fd_set rfds;
           struct timeval tv;
           int sel_ret;

           for (;;) {
               FD_ZERO(&rfds);
               FD_SET(STDIN_FILENO, &rfds);
               tv.tv_sec = STDIN_POLL_TIMEOUT_SEC;
               tv.tv_usec = 0;

               sel_ret = select(STDIN_FILENO + 1, &rfds, NULL, NULL, &tv);
               if (sel_ret < 0) {
                   /* select error (e.g., EINTR from signal) — treat as fatal */
                   goto cleanup;
               }
               if (sel_ret > 0) {
                   break;  /* stdin is readable — proceed to read_exact */
               }
               /* sel_ret == 0: timeout — loop back to poll again */
           }

           /* Read 5-byte header: [1 cmd][4 payload_len BE] */
           uint8_t hdr[5];
           if (read_exact(hdr, 5) != 0) {
               /* stdin closed — normal exit */
               break;
           }

           // ... rest of command dispatch unchanged ...
       ```

    4. Add a `cleanup` label before the existing `free(image_buf)` line:
       ```c
       cleanup:
           free(image_buf);
           DEV_ModuleExit();
           return 0;
       ```

    5. Change the existing `break` (stdin closed) to `goto cleanup` so both exit paths go through the same cleanup.

    The key insight: when the BEAM VM is killed, the pipe's write end closes, making stdin readable. `select()` returns immediately, then `read_exact()` returns -1 (EOF), and the port exits. Without `select()`, the port would block in `read()` inside `read_exact()` — but on Linux/macOS, a closed pipe unblocks `read()` anyway. The `select()` with timeout adds defense-in-depth: even if some OS edge case keeps the fd open, the port re-checks stdin readability every `STDIN_POLL_TIMEOUT_SEC` second(s).

    Do NOT modify: `read_exact()`, `write_exact()`, `send_ok()`, `send_error()`, or any command dispatch logic. The only change is the idle-wait pattern before reading the next command header.
  </action>
