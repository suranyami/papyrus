# Plan 05-01: SVG Backend with resvg

## Goal
Implement `Papyrus.Renderer.Headless` with SVG-based rendering using the `resvg` library — a lightweight, precompiled NIF (~3MB) that converts SVG to PNG without browser dependencies.

## Context
This is the **first backend** in the hybrid rendering approach. It targets embedded/Nerves deployments where installing Chrome or wkhtmltoimage is impractical. The `resvg` Hex package provides precompiled NIFs for Linux/macOS/Windows with no external dependencies.

**Tradeoffs:**
- ✅ 3MB dependency, works on Nerves
- ✅ Precompiled NIFs, zero runtime dependencies
- ❌ Limited CSS support (no flexbox, grid, complex layouts)
- ❌ HTML must be converted to simplified SVG

## Requirements Addressed
- **RENDER-04**: HTML to bitmap rendering — SVG backend path

## Pre-conditions
- `Papyrus.Bitmap.from_image/2` exists and accepts a file path + `DisplaySpec`
- `Papyrus.Display.display/2` exists and accepts a server + binary buffer
- `Papyrus.DisplaySpec` struct has `width`, `height`, `buffer_size` fields

### Task 2: Create Papyrus.Renderer.Headless module (SVG backend)

**Files to create/modify:**
- `lib/papyrus/renderer/headless.ex` — new module

**Implementation details:**

Create `Papyrus.Renderer.Headless` with:

1. **`render_html/2`** (SVG backend):
   - Signature: `render_html(input, spec)` where `input` is `{:html, String.t()} | {:url, String.t()} | {:file, String.t()}` and `spec` is `%DisplaySpec{}`
   - Returns `{:ok, binary()} | {:error, String.t()}`
   - First checks `Code.ensure_loaded?(Resvg)` — if false, returns `{:error, "resvg not available -- add {:resvg, \"~> 0.5\"} to your deps"}`
   - For `{:html, html}` input:
     - Call `html_to_svg/2` to convert HTML to simplified SVG
     - Call `render_svg/2` to render SVG to PNG binary via resvg
     - Pass PNG binary to `Papyrus.Bitmap.from_image/2`
   - For `{:url, url}` input:
     - Return `{:error, "URL input requires wkhtmltoimage backend — use render_html(input, spec, backend: :wkhtmltoimage)"}`
   - For `{:file, path}` input:
     - Read file content, treat as HTML or SVG file
     - Same pipeline as `{:html, html}`

2. **`render_svg/2`** (public, for direct SVG use):
   - Signature: `render_svg(svg_string, spec)`
   - Returns `{:ok, binary()} | {:error, String.t()}`
   - Calls `Resvg.svg_string_to_png_buffer/2` with width/height from spec
   - Passes result to `Papyrus.Bitmap.from_image/2`

3. **`display/3`** (convenience wrapper):
   - Signature: `display(input, display_pid, spec)`
   - Returns `{:ok, :sent} | {:error, String.t()}`
   - Calls `render_html/2`, then on success calls `Papyrus.Display.display(display_pid, bitmap_binary)`

4. **Private helpers:**
   - `html_to_svg/2` — converts simplified HTML to SVG format
     - Wraps content in `<svg width="W" height="H">` container
     - Converts basic elements: `<div>` → `<rect>`, `<p>/<h1>/<h2>` → `<text>`, `<img>` → `<image>` (with xlink:href)
     - Supports inline styles: `fill`, `font-size`, `font-weight`, `text-anchor`
     - Strips unsupported CSS (flexbox, grid, positioning)
   - `chromic_available?/0` — check `Code.ensure_loaded?(Resvg)`

**HTML-to-SVG conversion rules:**

| HTML | SVG Equivalent |
|------|----------------|
| `<h1>Text</h1>` | `<text x="16" y="40" font-size="32" font-weight="bold">Text</text>` |
| `<p>Text</p>` | `<text x="16" y="80" font-size="14">Text</text>` |
| `<div style="background:#f00">` | `<rect x="0" y="0" width="W" height="H" fill="#f00"/>` |
| `<img src="data:...">` | `<image x="0" y="0" width="W" height="H" href="data:..."/>` |
| `<span style="color:#abc">` | `<tspan fill="#abc">` (nested in parent text) |

**Acceptance criteria:**
- [ ] `lib/papyrus/renderer/headless.ex` exists with `render_html/2`, `render_svg/2`, `display/3`
- [ ] `mix compile --warnings-as-errors` passes
- [ ] Module has `@moduledoc`, `@doc`, `@spec` for all public functions
- [ ] `@since "0.2.0"` tag on module and public functions
- [ ] Temp file cleanup happens in `after` block when writing PNG intermediates
