# Plan 05-01: Implement Papyrus.Renderer.Headless Module

## Goal
Create the `Papyrus.Renderer.Headless` module with `render_html/2` and `display/3`, wire ChromicPDF as an optional dependency, and add conditional startup in `Papyrus.Application`.

## Context
This is the core implementation plan for Phase 5. It adds the headless HTML rendering capability that lets developers render HTML/URL/file content to ePaper displays via ChromicPDF. The module connects ChromicPDF's `capture_screenshot/2` output to the existing `Papyrus.Bitmap.from_image/2` pipeline through a temp-file bridge.

## Requirements Addressed
- **RENDER-04**: HTML to bitmap rendering via headless Chromium; optional `chromic_pdf` dependency

## Pre-conditions
- `Papyrus.Bitmap.from_image/2` exists and accepts a file path + `DisplaySpec` (confirmed in `lib/papyrus/bitmap.ex`)
- `Papyrus.Display.display/2` exists and accepts a server + binary buffer (confirmed in `lib/papyrus/display.ex`)
- `Papyrus.DisplaySpec` struct has `width`, `height`, `buffer_size` fields (confirmed in `lib/papyrus/display_spec.ex`)

## Tasks

### Task 1: Add chromic_pdf dependency and update Application supervision

**Files to create/modify:**
- `mix.exs` — add `{:chromic_pdf, "~> 1.17", optional: true}` to deps
- `lib/papyrus/application.ex` — add conditional ChromicPDF child start

**Implementation details:**

In `mix.exs`, add to the `deps/0` function:
```elixir
{:chromic_pdf, "~> 1.17", optional: true}
```

Do NOT add `:chromic_pdf` to `extra_applications` — the conditional startup handles this.

In `lib/papyrus/application.ex`, update `start/2` to conditionally include ChromicPDF as a child (per D-07):
```elixir
def start(_type, _args) do
  children =
    if Code.ensure_loaded?(ChromicPDF) do
      [{ChromicPDF, chromic_pdf_opts()}]
    else
      []
    end

  opts = [strategy: :one_for_one, name: Papyrus.Supervisor]
  Supervisor.start_link(children, opts)
end

defp chromic_pdf_opts do
  Application.get_env(:papyrus, :chromic_pdf_opts, [])
end
```

The `chromic_pdf_opts/0` helper allows consuming apps to configure ChromicPDF pool size, Chrome path, etc. via `config :papyrus, :chromic_pdf_opts, [...]`.

Run `mix deps.get` after updating mix.exs.

**Acceptance criteria:**
- [ ] `mix deps.get` succeeds and `chromic_pdf` appears in `mix.lock`
- [ ] `mix compile --warnings-as-errors` passes
- [ ] On a system without chromic_pdf compiled, `Papyrus.Application` starts with an empty children list (existing behavior preserved)

### Task 2: Create Papyrus.Renderer.Headless module

**Files to create/modify:**
- `lib/papyrus/renderer/headless.ex` — new module

**Implementation details:**

Create `Papyrus.Renderer.Headless` with:

1. **`render_html/2`** (per D-01, D-02, D-03, D-04, D-05, D-06):
   - Signature: `render_html(input, spec)` where `input` is `{:html, String.t()} | {:url, String.t()} | {:file, String.t()}` and `spec` is `%DisplaySpec{}`
   - Returns `{:ok, binary()} | {:error, String.t()}`
   - First checks `Code.ensure_loaded?(ChromicPDF)` — if false, returns `{:error, "chromic_pdf not available -- add {:chromic_pdf, \"~> 1.17\"} to your deps and ensure it is started"}` (per D-08)
   - Maps input to ChromicPDF source tuple via private `to_chromic_source/1`:
     - `{:html, html}` -> `{:html, html}` (passthrough)
     - `{:url, url}` -> `{:url, url}` (passthrough)
     - `{:file, path}` -> `{:html, File.read!(path)}` (read file, pass as HTML string — avoids file:// URL encoding issues)
   - For `{:html, _}` inputs, inject a CSS reset to constrain layout to display dimensions: wrap HTML in `<style>html,body{width:#{spec.width}px;height:#{spec.height}px;overflow:hidden;margin:0;padding:0;}</style>` prepended to the HTML content
   - Calls `ChromicPDF.capture_screenshot/2` with `capture_screenshot: %{"clip" => %{"x" => 0, "y" => 0, "width" => spec.width, "height" => spec.height, "scale" => 1}}` — use string keys for nested CDP parameters
   - Decodes Base64 result with `Base.decode64!(blob, ignore: :whitespace, padding: false)`
   - Writes decoded PNG to a temp file (`Path.join(System.tmp_dir!(), "papyrus_#{:erlang.unique_integer([:positive])}.png")`)
   - Passes temp file path to `Papyrus.Bitmap.from_image/2` in a `try/after` block that always deletes the temp file
   - Wraps the entire ChromicPDF call in `try/catch :exit, reason ->` to handle the case where ChromicPDF module is loaded but the pool is not running (Pitfall 5 from RESEARCH.md) — returns `{:error, "chromic_pdf process not running -- ensure ChromicPDF is started in your supervision tree"}`

2. **`display/3`** (per D-01):
   - Signature: `display(input, display_pid, spec)` where `display_pid` is a `GenServer.server()`
   - Returns `{:ok, :sent} | {:error, String.t()}`
   - Calls `render_html/2`, then on success calls `Papyrus.Display.display(display_pid, bitmap_binary)`, mapping `:ok` to `{:ok, :sent}`

3. **Typespecs** for both public functions. Add `@moduledoc` with `@since "0.2.0"` tag. Document that `chromic_pdf` is an optional dependency.

4. **Private helpers:**
   - `to_chromic_source/1` — pattern-match on tagged tuple, return ChromicPDF source
   - `screenshot_opts/1` — build CDP clip options from DisplaySpec
   - `with_temp_png/2` — write binary to temp file, call function, clean up in `after`
   - `inject_viewport_css/2` — for `{:html, _}` sources, prepend CSS reset style tag constraining to spec dimensions
   - `chromic_available?/0` — check `Code.ensure_loaded?(ChromicPDF)`, extracted to a named function so it can be overridden via `Application.get_env(:papyrus, :chromic_pdf_available)` for testing the "not available" code path

**Acceptance criteria:**
- [ ] `lib/papyrus/renderer/headless.ex` exists with `render_html/2` and `display/3` public functions
- [ ] `mix compile --warnings-as-errors` passes
- [ ] Module has `@moduledoc`, `@doc`, `@spec` for both public functions
- [ ] No bang functions in the happy path that could raise without rescue (Base64 decode is wrapped)
- [ ] Temp file cleanup happens in `after` block unconditionally
