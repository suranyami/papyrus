# Plan 05-02: Tests for Headless Renderer

## Goal
Create comprehensive ExUnit tests for `Papyrus.Renderer.Headless` covering all input types, error paths, and the display convenience wrapper — all runnable without Chromium installed.

## Context
The headless renderer depends on ChromicPDF which requires Chromium at runtime. Tests must mock the ChromicPDF interaction so they run in CI and on developer machines without a browser. The "chromic_pdf not available" path is tested by overriding the availability check via application config. The happy path is tested by mocking `ChromicPDF.capture_screenshot/2` to return a known Base64-encoded PNG.

## Requirements Addressed
- **RENDER-04**: HTML to bitmap rendering — test coverage for all input variants, error contract, and integration with bitmap pipeline

## Pre-conditions
- Plan 05-01 is complete: `Papyrus.Renderer.Headless` module exists with `render_html/2` and `display/3`
- `chromic_pdf` is in deps (optional)
- `Papyrus.Bitmap.from_image/2` works and has test fixtures available
- The `chromic_available?/0` function reads from `Application.get_env(:papyrus, :chromic_pdf_available)` as an override

## Tasks

### Task 1: Create test helper and test file for Headless renderer

**Files to create/modify:**
- `test/papyrus/renderer/headless_test.exs` — new test file

**Implementation details:**

Create `test/papyrus/renderer/headless_test.exs` with:

**Test setup:**
- Use `ExUnit.Case, async: false` (tests modify application env)
- Create a test `DisplaySpec` struct: `%Papyrus.DisplaySpec{model: :test, width: 100, height: 100, buffer_size: 1250, pin_config: %{rst: 6, dc: 13, cs: 8, busy: 5}, bit_order: :white_high}`
  - 100x100 pixels / 8 bits per byte = 1250 bytes buffer
- For the ChromicPDF mock: create a small valid PNG binary using `StbImage` (already in deps). In test setup, create a 100x100 white PNG via `StbImage.new(100, 100, 3, data: :binary.copy(<<255, 255, 255>>, 100 * 100))`, save to a fixture file, Base64-encode it, and use that as the mock return value.

**Test cases — "chromic_pdf not available" path:**

1. `test "render_html/2 returns error when chromic_pdf not available"` — Set `Application.put_env(:papyrus, :chromic_pdf_available, false)`, call `render_html({:html, "<h1>Test</h1>"}, spec)`, assert returns `{:error, msg}` where `msg` contains "chromic_pdf not available". Clean up in `on_exit`.

2. `test "display/3 returns error when chromic_pdf not available"` — Same override, call `display({:html, "<h1>Test</h1>"}, self(), spec)`, assert `{:error, _}`.

**Test cases — input dispatch (unit tests for private helpers via public API):**

Since ChromicPDF may or may not be installed in the test environment, these tests should use a mock approach. The recommended approach:

- Extract the ChromicPDF call into a configurable function: `Application.get_env(:papyrus, :chromic_pdf_capture, &ChromicPDF.capture_screenshot/2)`. In tests, replace with a function that records its arguments and returns a known Base64 PNG.
- Alternative: If `Code.ensure_loaded?(ChromicPDF)` is true in the test env (chromic_pdf is compiled as a dep), use Mox. Define a `Papyrus.ChromicPDF.Behaviour` with `capture_screenshot/2` callback, and inject the implementation.

The simplest approach that works without Mox: make the renderer call a configurable capture function. In production it defaults to `&ChromicPDF.capture_screenshot/2`. In tests, inject a mock function via application env.

If this design change is too invasive, an alternative: test input dispatch by testing `to_chromic_source/1` directly if it is made `@doc false` public, or test through the full path with a real ChromicPDF (gated behind `@moduletag :requires_chromium`).

**Recommended: Use the configurable capture function approach.** Update `lib/papyrus/renderer/headless.ex` to read the capture function from config:
```elixir
defp capture_screenshot(source, opts) do
  capture_fn = Application.get_env(:papyrus, :chromic_pdf_capture, &ChromicPDF.capture_screenshot/2)
  capture_fn.(source, opts)
end
```

Then test cases:

3. `test "render_html/2 with {:html, _} passes HTML source to ChromicPDF"` — Inject a mock capture function that asserts it receives `{:html, _}` as first arg, returns `{:ok, base64_png}`. Assert `render_html({:html, "<h1>Hi</h1>"}, spec)` returns `{:ok, binary}` where `byte_size(binary) == spec.buffer_size`.

4. `test "render_html/2 with {:url, _} passes URL source to ChromicPDF"` — Mock receives `{:url, "http://example.com"}`, returns base64 PNG. Assert success.

5. `test "render_html/2 with {:file, _} reads file and passes as {:html, _}"` — Create a temp HTML file, call with `{:file, path}`. Mock asserts it receives `{:html, content}` where `content` contains the file's text. Clean up temp file in `on_exit`.

6. `test "render_html/2 passes correct clip dimensions from DisplaySpec"` — Mock inspects the opts argument, asserts `capture_screenshot` map has `"clip"` with `"width" => 100, "height" => 100`.

**Test cases — error handling:**

7. `test "render_html/2 returns error when ChromicPDF capture fails"` — Mock returns `{:error, "chrome crashed"}`. Assert `render_html/2` returns `{:error, "chrome crashed"}`.

8. `test "render_html/2 returns error when ChromicPDF process not running"` — Mock raises an exit: `exit(:noproc)`. Assert returns `{:error, msg}` containing "not running".

9. `test "render_html/2 with {:file, _} returns error for missing file"` — Call with `{:file, "/nonexistent/path.html"}`. Assert returns `{:error, _}` (File.read! will raise; the function should rescue or the test documents the expected behavior).

**Test cases — temp file cleanup:**

10. `test "render_html/2 cleans up temp PNG file after success"` — After a successful call, verify no `papyrus_*.png` files exist in `System.tmp_dir!()` that were created during the test.

11. `test "render_html/2 cleans up temp PNG file after bitmap error"` — Mock returns valid Base64 PNG but corrupt image data that `Bitmap.from_image/2` rejects. Verify temp file is cleaned up.
