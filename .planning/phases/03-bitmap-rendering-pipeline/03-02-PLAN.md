**lib/papyrus/bitmap/resize.ex** — `Papyrus.Bitmap.Resize` module:

```elixir
defmodule Papyrus.Bitmap.Resize do
  @moduledoc "Letterbox resize: scale to fit display dimensions, pad with white."

  @doc """
  Resize an StbImage to fit within target_w x target_h using letterbox strategy.
  Uses StbImage.resize/3 for the scaling (Mitchell/Catmull-Rom, per D-06 quality intent).
  Pads remaining space with white (255) pixels.
  Returns a flat binary of exactly target_w * target_h grayscale bytes.
  """
  @spec letterbox(StbImage.t(), pos_integer(), pos_integer()) :: binary()
  def letterbox(img, target_w, target_h)
```

Implementation:
1. Extract `{img_h, img_w, 1} = img.shape`
2. If `img_w == target_w and img_h == target_h`, return `img.data` directly
3. Calculate scale: `scale = min(target_w / img_w, target_h / img_h)`
4. Calculate `scaled_w = trunc(img_w * scale)` and `scaled_h = trunc(img_h * scale)` — ensure both are >= 1
5. Call `StbImage.resize(img, scaled_h, scaled_w)` to get resized image
6. Calculate padding: `pad_top = div(target_h - scaled_h, 2)`, `pad_bottom = target_h - scaled_h - pad_top`, `pad_left = div(target_w - scaled_w, 2)`, `pad_right = target_w - scaled_w - pad_left`
7. Build output binary row by row:
   - `pad_top` rows of `target_w` bytes of 255 (white)
   - For each of `scaled_h` rows: `pad_left` bytes of 255 + row data from resized image + `pad_right` bytes of 255
   - `pad_bottom` rows of `target_w` bytes of 255
8. Assert output byte_size == `target_w * target_h`, return the binary

Use `:binary.copy(<<255>>, n)` for white padding segments. Extract rows from resized image data using `binary_part(resized.data, row_idx * scaled_w, scaled_w)`.

**lib/papyrus/bitmap/pack.ex** — `Papyrus.Bitmap.Pack` module:

```elixir
defmodule Papyrus.Bitmap.Pack do
  @moduledoc "Grayscale to 1-bit conversion: threshold, Floyd-Steinberg dithering, MSB-first bit packing."

  @doc """
  Convert grayscale pixel data to a packed 1-bit buffer.
  - pixels: binary of width*height grayscale bytes
  - width: pixel width of image
  - bit_order: :white_high or :white_low
  - opts: [dither: true] for Floyd-Steinberg
  Returns a binary of ceil(width/8) * height bytes.
  """
  @spec to_buffer(binary(), pos_integer(), Papyrus.DisplaySpec.bit_order(), keyword()) :: binary()
  def to_buffer(pixels, width, bit_order, opts \\ [])
```

Implementation:

**Threshold path (default, D-08):**
1. Determine `white_bit`: if `bit_order == :white_high`, white_bit = 1, else 0 (per D-10)
2. Convert binary to list of pixel values: `for <<b <- pixels>>, do: b`
3. Chunk into rows of `width` pixels
4. For each row, chunk into groups of 8 (padding last chunk with white_bit if width not multiple of 8)
5. Pack each group MSB-first: `<<p0::1, p1::1, p2::1, p3::1, p4::1, p5::1, p6::1, p7::1>>`
6. Where each `pN` = if `pixel_value > 128`, white_bit, else `1 - white_bit`
7. Concatenate all row binaries

**Floyd-Steinberg path (opts dither: true, D-09):**
1. Convert pixel binary to a list of integers
2. Build a mutable-style map `%{{x, y} => error_accumulation}` or process row-by-row with error carry
3. Scan left-to-right, top-to-bottom:
   - `old_val = clamp(pixel_value + accumulated_error, 0, 255)` — clamp BEFORE threshold
   - `new_val = if old_val > 128, do: 255, else: 0`
   - `quant_err = old_val - new_val`
   - Distribute error: `{x+1, y} += quant_err * 7/16`, `{x-1, y+1} += quant_err * 3/16`, `{x, y+1} += quant_err * 5/16`, `{x+1, y+1} += quant_err * 1/16`
   - Use integer arithmetic to avoid float: `div(quant_err * 7, 16)` etc.
4. After dithering, the pixel values are 0 or 255 — pack using same MSB-first logic as threshold path

Recommended approach for Floyd-Steinberg in Elixir: process row by row using `Enum.reduce/3`. For each row, carry forward a `current_row_errors` list (errors from left neighbors in this row) and a `next_row_errors` list (errors propagated to the row below). This avoids a full `%{{x,y} => val}` map.

Row byte width: `row_bytes = div(width + 7, 8)` (equivalent to `ceil(width/8)` using integer math).

**Update lib/papyrus/bitmap.ex** — replace the stub `from_image/2` and `from_image/3` with the real pipeline:

```elixir
def from_image(path, spec), do: from_image(path, spec, [])

def from_image(path, %DisplaySpec{} = spec, opts) do
  with {:ok, img} <- loader().load(path) do
    pixels = Papyrus.Bitmap.Resize.letterbox(img, spec.width, spec.height)
    buffer = Papyrus.Bitmap.Pack.to_buffer(pixels, spec.width, spec.bit_order, opts)
    {:ok, buffer}
  end
end
```

This is the complete pipeline: load -> resize/letterbox -> threshold/dither + pack.

Key points:
- `from_image/2` delegates to `from_image/3` with empty opts (per D-08, default is simple threshold)
- `from_image/3` accepts `dither: true` option (per D-09), passed through to Pack.to_buffer
- Errors from loader propagate automatically via `with` (per D-07, no dimension mismatch error)
- The `loader()` call uses the configurable backend (per D-03)
